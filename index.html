Algorithm Developmenet Process :

Description :

Creating algorithms should follow basic steps:

  1. Understand the Problem
  2. Decide On:
    • Computational means
    • Approximate Solving or Exact Solution
    • Data Structures
    • Algorithm Design Techniques
  3. Design Algorithm - Use mainly Pseudo Code
  4. Prove Correctness
  5. Analyze the Algorithm - Time complexity and Space complexity
  6. Code the Algorithm

Each step in the process should be achieved in order. However, achieving the simplest soltuion is still considered an algorithm even if its worst case

Big O Time Complexities : common

Smallest to Larger

  • O(1) : Constant Time
  • O(log n) : Logarithmic
  • O(n log n) : N Logarithmic n
    • - Looping through n items and within looping n/2 items
  • O(n^2) : Quadratic
  • O(n!) : Factorial

Two-Pointer Step - Multi-approach :

Description :

Traversing sorted lists allows for bi-directional incrementing. Two-pointers allows for array like structures to be travered in half (n/2) the time.

Problems Solved :

Increases performance time of linear structrue traversal.

Formatting :

  1. Sort the array structure if required.
  2. Create two pointers. - ex. int position1, position2;
  3. Increment and decrement the appropriate pointer in the loop.

Description :

Problems Solved :
