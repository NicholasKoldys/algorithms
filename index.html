Algorithm Development Process :

Description :

An algorithm is a series of defined instructions that perform a task.

Assumptions on the underlying data can have an effect on speed and efficiency.

Normalized data will allow algorithms to be used for many input types.

Creating algorithms should follow basic steps:

  1. Understand the Problem
  2. Decide On:
    • Computational means
    • Approximate Solving or Exact Solution
    • Data Structures
    • Algorithm Design Techniques
  3. Design Algorithm - Use mainly Pseudo Code
  4. Prove Correctness
  5. Analyze the Algorithm - Time complexity and Space complexity
  6. Code the Algorithm

Each step in the process should be achieved in order. However, achieving the simplest soltuion is still considered an algorithm even if its worst case

Assumptions :

Optimized and Efficient Algorithms don't come from data structure use or leet programming skills, only from Informed Assumptions.

Performing brute force methods and naive approaches to problems are not incorrect, but can result in MESSY and INEFFICIENT code.

Assumptions = means to guess or assume that the data is following a certain derived rule.

For Example, if there are three digits and we need to determine which is the median, we can make a guess that one of the digits is the maximum --- we can always correct false assumptions, as we are attempting to solve the solution --- this maximum can either be TRUE or FALSE, if it's TRUE we have one less step, if it's FALSE we would have already spent a step checking it anyway, if we had not selected a maximum.

Big O Time Complexities : common

Smallest to Larger :

  • O(1) : Constant Time
  • O(log n) : Logarithmic
  • O(n log n) : N Logarithmic n
    • - Looping through n items and within looping n/2 items
  • O(n^2) : Quadratic
  • O(n!) : Factorial

Recursion-Basics - Process to forming a recursive loop :

Description :

Recursive functions call on themselves to iterate through a process before reaching the outcome.

Problems Solved :

Simple to read code, without if else branches that lead to other functions.

Basic Rules to follow :

  1. Always return, unless acting by reference on the datastructure.
  2. Always condition before stepping into recursion.
  3. Recursion should only act on the given parameters.
